# 概念板块偏度扫描器 - 设计方案

## 一、项目概述

### 1.1 目标
基于本地 DuckDB 数据库（收盘后更新），对全市场 **270个概念板块指数** 进行波动率偏度扫描，帮助用户：
- 全面了解各概念板块的走势特征（上行/下行波动率对比）
- 筛选出偏度比优异（≥1）的板块，发现强势热点
- 快速定位偏度比差（<1）的板块，规避弱势板块

### 1.2 数据特征
- **非实时数据**：每天收盘后更新一次
- **数据源**：`/Users/jiashen/WorkBuddy/Claw/tradingview/dbs/tdx.db`（DuckDB）
- **板块数据**：
  - 概念板块：270个（`raw_tdx_blocks_info` 表，`block_type = 'concept'`）
  - 板块日线：存储在 `raw_stocks_daily` 表，与个股数据同表
  - 最新数据日期：2026-04-23
  - 历史数据范围：2005年 ~ 至今（不同板块起始时间不同）

### 1.3 现有架构参考
- 已有 `SkewScannerPage.tsx` 组件（个股偏度扫描器）
- 已有 `calculateVolatility` 函数（`src/utils/stockData.ts`）
- 偏度比定义：`volSkew = upVolatility / downVolatility`
  - `≥1`：上行波动主导，强势特征
  - `<1`：下行波动主导，弱势特征

## 二、整体架构设计

### 2.1 数据流

```
DuckDB (tdx.db)
    ↓
Python 预计算脚本 (scan-sector-skew.py)
    ↓
生成 JSON 文件 (public/sector-skew-data.json)
    ↓
React 前端 (SectorSkewScannerPage)
    ↓
排序 / 筛选 / 可视化
```

### 2.2 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 数据层 | DuckDB (Python) | 本地数据库查询 |
| 预计算 | Python 脚本 | 手动运行，生成 JSON 文件 |
| 前端 | React + TypeScript | 新建扫描页面组件 |
| 缓存 | localStorage | 缓存扫描结果，避免重复读取 |

### 2.3 为什么采用预计算方案

- 现有 API 服务在其他项目中，不应修改
- 前端无法直接读取本地 DuckDB 文件
- 预计算脚本简单高效，查询 + 计算一次性完成
- 页面提供手动触发按钮，用户可随时重新扫描

### 3.1 预计算脚本

`scripts/scan-sector-skew.py`

#### 运行方式
```bash

```

#### 脚本功能
1. 连接 `dbs/tdx.db` 数据库
2. 查询所有概念板块信息

### 3.3 核心计算逻辑

#### 3.3.1 SQL 查询（Python 脚本执行）

```sql
-- 获取所有概念板块
SELECT bi.block_symbol, bi.block_name, bi.block_code
FROM raw_tdx_blocks_info bi
WHERE bi.block_type = 'concept'
ORDER BY bi.block_symbol;

-- 获取板块日线数据（最近N条，N = calcWindow + 10）
SELECT date, open, high, low, close, volume, amount, change_pct
FROM raw_stocks_daily
WHERE symbol = 'sh880520'
ORDER BY date DESC
LIMIT 50;
```

#### 3.3.2 偏度计算（Python 脚本）

复用现有的波动率计算逻辑：
1. 从日线数据计算每日收益率
2. 分离上行收益率（>0）和下行收益率（<0）
3. 计算上行波动率（上行收益率标准差）和下行波动率（下行收益率标准差）
4. 偏度比 = 上行波动率 / 下行波动率

### 3.4 前端扫描按钮逻辑

用户点击"执行扫描"按钮后：
1. 从 `/sector-skew-data.json` 读取预计算数据
2. 解析 JSON，计算统计指标（平均偏度、强势板块数等）
3. 渲染表格，支持排序和筛选
4. 将结果缓存到 localStorage
5. 显示扫描时间和数据新鲜度提示

## 四、实现方案

### 预计算脚本 + 前端展示（最终方案）

**工作流程**：
1. 用户手动运行 Python 脚本，扫描数据库生成 JSON 文件
2. 前端页面提供"手动触发扫描"按钮，点击后读取 JSON 文件
3. 前端解析数据，展示结果，支持排序和筛选

**优点**：
- 不修改现有 API 服务
- 查询计算在 Python 端完成，高效
- 前端只负责展示，逻辑简单
- 手动触发，用户可控

## 九、风险与注意事项

1. **性能**：270个板块的日线数据查询和计算需要一定时间，需加进度提示
2. **数据一致性**：确保前端显示的板块名称与数据库一致
3. **错误处理**：部分板块可能数据不足（如新板块），需优雅处理
4. **浏览器兼容**：DuckDB 的 WASM 版本可在浏览器运行，但首次加载较大，不推荐
5. **数据更新**：明确告知用户这不是实时数据，每天收盘后更新
